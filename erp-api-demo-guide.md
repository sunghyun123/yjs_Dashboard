# ERP UI 시연 자료 (개발사 전달용)

## 1) 파일 목적
- 파일명: `erp-api-demo.html`
- 목적: 백엔드 구현 전, 프론트엔드 화면/트리거/UI 흐름을 먼저 합의하기 위한 시연 자료
- 범위: 로그인, 메인 메뉴, 전송 데이터 등록, ERP 정보 조회 화면

## 2) 실행 방법
- 전달 받은 `erp-api-demo.html` 파일을 브라우저에서 실행
- 권장 전달 방식: `zip` 압축 후 전송 (`erp-api-demo.zip`)
- 참고: 현재 화면은 UI/흐름 시연용이며 실제 ERP 서버 호출은 연결 전 상태

## 3) 화면 구성 (개발사 요청 반영)
- 로그인 화면(1차 인증): 아이디/비밀번호 입력 후 로그인
- 메인 메뉴: 아래 2개 버튼으로 구성
  - `전송 데이터 등록`
  - `ERP 정보 조회`
- 전송 데이터 등록 화면:
  - 전송 구분 콤보박스
  - 엑셀 파일 선택
  - 전송 버튼
  - 작업일지 이미지 업로드 + AI 추출 실행 버튼
  - 추출값 폼 자동 채움 버튼
  - 투입실적 표(여러 행 입력)
  - 전송 결과 메시지(성공/실패)
- ERP 정보 조회 화면:
  - 지중 No 선택
  - 공정률/누적기성 표시
  - 요청/응답 JSON 미리보기

## 4) 트리거/처리 흐름
- 전송 흐름:
  1. 사용자 입력(직접 입력 또는 엑셀 선택)
  2. 사용자 전송 버튼 클릭
  3. 배열 단위 POST 요청 (여러 줄 일괄 전송)
  4. 백엔드 처리 결과를 프론트 메시지로 표시
- 작업일지 자동화 흐름:
  1. 사용자가 시공팀 작업일지 사진 업로드
  2. `AI 추출 실행` 클릭
  3. 백엔드 AI 추출 API 호출 (`POST /api/vision/extract-worklog`)
  4. 추출된 `records[]`를 `추출값 폼 반영` 버튼으로 자동 입력
  5. 사용자가 확인 후 전송 버튼 클릭
- 조회 흐름:
  1. ERP 정보 조회 화면 진입 시 자동 GET
  2. 지중 No 변경 시 자동 GET 재호출
  3. 수동 재조회 버튼 클릭 시 GET

## 5) 요청 데이터 예시
- 투입실적 등록(POST 예시)
```json
{
  "transfer_type": "input_record",
  "request_time": "2026-04-22T03:12:00.000Z",
  "records": [
    {
      "work_code": "SY26-005",
      "input_date": "2026-04-22",
      "item_name": "상용직",
      "day_value": 8,
      "night_value": 2
    },
    {
      "work_code": "SY26-005",
      "input_date": "2026-04-22",
      "item_name": "크레인",
      "day_value": 1,
      "night_value": 0
    }
  ]
}
```

- 공정률 조회(GET 예시)
```text
GET /erp/project-progress?work_code=SY26-005
```

- 작업일지 이미지 추출(POST 예시)
```text
POST /api/vision/extract-worklog
Content-Type: multipart/form-data
file=<작업일지 이미지>
```

## 6) 역할 분담 정리
- 프론트(UI) 담당: 화면 구성, 입력 트리거, 결과 메시지 노출
- 백엔드(개발사) 담당: 파일 파싱, DB Insert/조회, 실제 API 구현
- 결론: 현재 파일은 역할 분담 기준에 맞춘 프론트 시연 자료
